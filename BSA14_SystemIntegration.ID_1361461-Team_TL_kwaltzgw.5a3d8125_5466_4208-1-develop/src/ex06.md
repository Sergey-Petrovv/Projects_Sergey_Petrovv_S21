# Exercise 06 — Определение частоты и сроков для потоков данных

Для каждого потока задачи 2 определите:

1. Желаемая частота передачи;
2. Желаемые сроки доставки;
3. Максимально и минимально возможная частота передачи;
4. Максимально и минимально возможные сроки доставки.


# Определение частоты и сроков для потоков данных

Принятые допущения:

1. Канал интеграции: API/очередь (асинхронно где можно).
2. “Срок доставки” = время от отправки сообщения источником до доступности данных у получателя.
3. Для курьерских операций “оперативно” = секунды, для бухгалтерии допустимы минуты/часы.
4. Потоки, влияющие на работу курьера/выполнение заказа (требуют событийной передачи и секундных SLA (“оперативно”).
5. Финансовые потоки допустимо делать асинхронно и пакетно, SLA — минуты/часы (вплоть до суток в аварийном режиме).
6. Для офлайн-режима мобильного приложения уместно указывать “минимально возможную частоту” как синхронизация раз в N минут и “максимальные сроки” до 30 минут.
