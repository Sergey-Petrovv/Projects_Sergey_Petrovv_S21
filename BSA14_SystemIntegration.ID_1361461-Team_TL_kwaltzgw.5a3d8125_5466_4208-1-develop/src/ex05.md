# Определение требований к объему и количеству для потоков данных

Принятые допущения:

1. DF-01 и DF-07 в среднем коррелируют: сколько заказов создаём — примерно столько же строк уходит в бухучёт 
2. DF-04 обычно самый “шумный” поток, потому что статусы генерируются на каждый заказ несколько раз (NEW → PICKED_UP → DELIVERED/FAILED), плюс возможны повторы/ретраи.
3. DF-03 “тяжелее” по размеру: это не один заказ, а список.
4. DF-02 и DF-09 — типично пакетные/оконные нагрузки (импорт справочника и начисления).
